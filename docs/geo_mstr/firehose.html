Firehose log — the press-named gems, week by week

How the simplified solution works:
  1. 📰 Read the firehose. Each week, scan the news for tickers the press explicitly NAMES as thesis-driven movers — the journalists do the gem-discovery (CNBC/ETF.com named BWET weeks before it tripled).
  2. 🟢 Enter / hold while LIVE. A name stays on the watchlist while its driving thesis is live (war on, chokepoint shut).
  3. ⚪ Exit on decay. Drop it when the thesis resolves (ceasefire, Hormuz reopens).
  4. Mechanical sizing downstream (mean-variance optimizer; the LLM never sets weights).
No causal decision-tree — just the firehose, an entry/exit switch, and the optimizer. This is the event-first agent over a realistic GDELT firehose (BWET seeded); the live forward log is built by src/forward.py.
