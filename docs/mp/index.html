Plot 2 — Cumulative $ gain per agent (event)

Each funded event's running $ contribution to the book — a line climbs while the agent holds, then flatlines at its realized gain once it exits. ▲ marks the week the agent went live, ✕ its exit. The bold Total is the portfolio's gain (the lines sum to it). Flat-near-zero = an agent that contributed little; never-funded agents are omitted.

Plot 3 — Allocation over time

Capital committed per ticker (cash fills the rest). Fully invested while the watchlist is non-empty; to cash when the press names nothing live.

Plot 4 — Holdings timeline (proposed vs funded)

One row per ticker the curator named. Thin gray, small dots = proposed (on the live watchlist); thick colored, large dots = funded (the optimizer actually bought it).

Plot 5 — Dollars held per ticker

Capital in dollars per ticker over time (cash fills to the portfolio total, so the stack's top edge is the portfolio value). Plot 3 shows the same split as percentages.

Plot 6 — Cumulative $ gain per holding

Total dollar P&L each holding contributed over the window (Σ daily position-value × daily return). Green = winner, red = loser; the bars sum to the portfolio's total gain.

Plot 7 — Cumulative $ earned per agent (event)

Total dollar P&L attributed to each distinct agent (event id), partitioning a ticker's gain across its agents by their active windows — so a ticker that spawned two agents (e.g. BWET's ev2 then ev6) shows each one's own contribution. Green = winner, red = loser; the bars sum to the portfolio's total gain.

Plot 8 — Conviction score over time, per agent

Each agent's catalyst-conviction rating (1-10) week by week — how strong / early / datable it judged its own catalyst. The dashed grey line is the always-on SPY agent (spy_agent_conviction): a live event must out-rank it to be held.

Plot 9 — Gain vs conviction over time, per agent

Each agent's time-history as a connected path through (conviction x, cumulative $ gain y) space — one dot per week, joined in time order. A path that climbs up-and-right = conviction and gain rising together (a good, self-aware thesis); a path high on x but sinking on y = high conviction that isn't paying off (a conviction miss). SPY (the always-on agent) is the dashed grey path.

Green = winning thesis, red = losing; the headline is the precision (share of agents that made money).

Plot 11 — Watchlist by date

Each row is a date the live watchlist (or its funding) changed — the names the press kept thesis-live that week. Bold + colored = actually funded by the optimizer; gray = on the watchlist but pruned by the sizing floor.

EVERY week the firehose was scanned, and what the agent did with this gem: pre = scanned but not yet flagged; live / exit = held / thesis called dead; post = dropped, watching it stay dead. Shows the agent's reaction to the firehose before, during, and after the event — not just the live span.

Plot 13 — Agent journal — week-by-week (per event)

Each event-agent's arc since entry — one collapsible block per ticker (gem first), captioned with the event thesis. Use it to confirm what event/ticker was discovered, when, and that it exited as the thesis decayed.
